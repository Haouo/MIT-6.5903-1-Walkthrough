## 一句話總結（TL;DR）

本課程用來描述 DNN 運算的記法是 **Einsum**——一個緊湊的代數表達式，命名每個張量（tensor）運算元、每個**秩（rank）**（索引），以及算術運算子，但刻意對運算的執行順序隻字不提。本講透過**操作定義（Operational Definition for Einsums, ODE）**精確定義 Einsum，說明秩變數（rank variable）如何編碼被收縮（contracted）與未收縮（uncontracted）的維度，並示範**每一種典型 DNN 層**——矩陣–向量乘、矩陣–矩陣乘、全連接（FC）、2-D 卷積（CONV）——都可以表達為一個 Einsum，並最終化歸為矩陣乘法。下半場轉向 **Transformer 注意力機制（attention）**：用 Einsum 逐步拼出 Q/K/V 計算鏈，讓自注意力（self-attention）的結構如矩陣乘法般透明。核心硬體洞察是：*Einsum* 是 TeAAL 金字塔頂端的「Compute」規格——它回答「算什麼（what）」，卻把「怎麼算（how）」（映射 mapping、資料流 dataflow、分塊 tiling）完全留給後續講次。

## 學習目標（Learning Objectives）

讀完本講後，你應該能夠：

- 陳述 **Einsum 的操作定義（ODE）**，並用它評估任意 Einsum。
- 區分 Einsum 中的**收縮秩（contracted rank）**與**未收縮秩（uncontracted rank）**，辨識何者決定歸約（reduction）、何者決定迭代。
- 認識到對**同一運算**的 Einsum 可以有多種等價寫法（不同的索引字母、不同的秩排列順序）。
- 描述**分割（partitioning）**（將一個秩拆分為兩個）與**扁平化（flattening）**（將兩個秩合併為一個）作為互逆操作、且不改變底層運算的含義。
- 寫出 **1-D 卷積**、**2-D CONV 層**與 **FC 層**的 Einsum，並將任一者轉換為矩陣乘法。
- 追蹤**基本自注意力**的七個 Einsum 步驟（嵌入 → Q/K 投影 → QK 乘積 → softmax → V 投影 → AV 乘積 → 輸出投影）。
- 透過新增 H 秩，將單頭注意力擴展為**多頭注意力（Multi-Head Attention, MHA）**。
- 解釋注意力的 MAC 數量為何與序列長度 M **二次方縮放（quadratic scaling）**。

## 第一章 — Einsum：DNN 運算的精確語言

> *投影片：L04-1 … L04-19*

### Einsum 的操作定義（ODE）

本講以一個引例開場：

$$Z_{m,n} = A_{m,k} \times B_{n,k}$$

這就是一個 **Einsum**。每個下標都是一個**秩變數（rank variable）**（索引，在某個維度上取值）。本講透過 **Einsum 操作定義（Operational Definition for Einsums, ODE）** 定義求值方式：

1. 遍歷所有合法秩變數值所構成的空間中的**每一個點**——即**迭代空間（iteration space）**。
2. 在每個點，用各運算元在指定秩坐標處的值計算右側表達式。
3. 將結果**賦值**給左側運算元在指定秩坐標處——若該元素已非零，則**歸約**（累加）進去。

![Einsum ODE——範例 Einsum 與操作定義](../../assets/L04/L04-p04-ode-example-einsum.png)

ODE 刻意與執行順序無關。它告訴你「算什麼」（迭代空間每個點的結果），卻不規定運算的先後序列。這種自由度正是關鍵所在：同一個 Einsum 可以用任何合法的迴圈嵌套（loop nest）來求值，而不同的迴圈嵌套對應不同的硬體**資料流（dataflow）**——這正是 L05–L06 的主題。

### 收縮秩與未收縮秩

在矩陣–矩陣乘法 `Z_{m,n} = A_{m,k} × B_{k,n}` 中：

- `k` 只出現在右側，不出現在左側。它是**收縮秩**：其值被加總（歸約），以生成每個輸出元素。
- `m` 和 `n` 同時出現在兩側。它們是**未收縮秩**：每個不同的 (m, n) 組合產生一個獨立的輸出元素。

這個區分至關重要。收縮秩決定每個輸出點的算術工作量；未收縮秩決定輸出張量的形狀。

### 張量引用（Tensor References）與秩命名

投影片引入了一套記法，可獨立指定**秩名（rank name）**與**秩形狀（rank shape）**，而不依賴索引字母。張量 `A_{k,m}` 的秩名為「K」、「M」，秩形狀為 K、M，也可寫成 `A^{K,M}_{k,m}` 以示明確。關鍵是：**下標中秩變數的排列順序不影響 Einsum 所計算的內容**——變數與秩的配對是位置性的，但迭代空間是所有維度的笛卡兒積，與列出順序無關。

### Einsum 模式：一張投影片收盡 DNN 全部層型

投影片 8 整理了主要的 Einsum 模式：

| Einsum | 運算 |
|---|---|
| `Z_{m,n} = A_{m,k} × B_{k,n}` | 矩陣–矩陣乘（k 被收縮） |
| `Z_m = A_{k,m} × B_k` | 矩陣–向量乘 |
| `Z_{m,n} = A_m × B_n` | 笛卡兒積（外積） |
| `Z_m = A_m × B_m` | 逐元素乘法 |
| `Z_m = A_m + B_m` | 逐元素加法（算子為 + 而非 ×） |

核心觀察：秩變數的**名字**無關緊要（`Z_{p,q} = A_{p,r} × B_{q,r}` 仍然是矩陣–矩陣乘）。重要的是哪些變數被共享、哪些只出現在右側（被收縮）、哪些出現在兩側（未被收縮）。

![Einsum 模式——矩陣–矩陣乘及其變體](../../assets/L04/L04-p08-einsum-patterns.png)

### 秩變數模式：元組作為單一索引

當兩個或多個秩變數總是以坐標對的形式共同出現——如 `Z_{i,j} = A_{i,j} × B_{i,j}`——可以將其扁平化為單一複合索引 `ij = i × J + j`，得到 `Z_{ij} = A_{ij} × B_{ij}`。這純粹是記法上的便利；迭代空間完全相同。

### 分割（Partitioning）：將一個秩拆成兩個

給定 `Z_i = A_i × B_i`，若令 `i = i1 × I0 + i0`，則得到：

$$Z_{i1,i0} = A_{i1,i0} \times B_{i1,i0}$$

這是一個**分割後的 Einsum**。原始向量索引 `i` 被拆分為粗粒度索引 `i1` 和細粒度索引 `i0`。運算完全相同，只是改變了標記迭代空間元素的方式。**分割總是新增一個秩**；扁平化總是移除一個秩。兩者互為逆運算。

![分割張量視覺化——向量 A 沿 i1、i0 切分成 tile](../../assets/L04/L04-p12-partitioned-tensor.png)

> **為什麼重要：** 分割是**分塊（tiling）**的代數基礎——分塊是一種映射技術（L05–L06），它把大型運算拆成能裝入小型緩衝區的塊。Einsum 記法讓分塊的表達既明確又正確。

## 第二章 — 卷積的 Einsum 表達

> *投影片：L04-15 … L04-18*

### 1-D 卷積

最簡單的情況是一維卷積，有輸入 `I`、濾波器 `F` 和輸出 `O`：

$$O_q = I_{q+s} \times F_s$$

輸出秩 `q`（輸出位置）是未收縮的；濾波器秩 `s`（濾波器內的位置）是被收縮的。輸入在**移位索引** `q+s` 處被存取——這正是卷積滑動視窗（sliding window）本質的體現。

加入秩形狀後，表達式變為：

$$O_q^Q = I_{q+s}^W \times F_s^S$$

其中 Q = W − S + 1（有效卷積）。

![1-D 卷積——濾波器權重、輸入、輸出，以及 Einsum O_q = I_{q+s} × F_s](../../assets/L04/L04-p15-convolution-1d.png)

秩形狀投影片（投影片 18）還引入一個通用圖形範例：鄰接矩陣 `G^{S=6,D=6}`——展示 Einsum 可以同等自然地描述圖操作與卷積。

> **為什麼重要：** `q+s` 的索引依賴關係（托普利茲 Toeplitz 結構）正是卷積與普通矩陣乘法的根本區別。在第三章把 CONV 化歸為矩陣乘時，關鍵步驟就是「拆解」這個托普利茲關係。

## 第三章 — 核心運算：FC 與 CONV 化歸矩陣乘法

> *投影片：L04-20 … L04-172*

這是本投影片組篇幅最長的章節，由大量遞增動畫驅動。核心思想是：FC 層與 CONV 層都可以透過**秩扁平化**在代數上化歸為矩陣乘法——對硬體執行方式有深遠影響。

### 全連接層（FC）作為 Einsum 與矩陣–向量乘

FC 層將形狀為 (C, H, W) 的輸入激活映射到 M 個輸出神經元。其 Einsum 為：

$$O_m = I_{c,h,w} \times F_{m,c,h,w}$$

（`c`、`h`、`w` 為被收縮的秩；`m` 為未收縮秩。）對應的迴圈嵌套實作：

```
for m in [0, M):
  o[m] = 0
  for c in [0, C):
    for h in [0, H):
      for w in [0, W):
        o[m] += i[c][h][w] * f[m][c][h][w]
```

若將三個輸入秩 C、H、W **扁平化**為單一複合秩 `chw = H×W×c + W×h + w`，Einsum 即成為：

$$O_m = I_{chw} \times F_{m,chw}$$

這正是**矩陣–向量乘**（濾波器：M×CHW；輸入：CHW×1；輸出：M×1）。

加入 N 個輸入樣本的批次（batch），輸入變為 I_{n,chw}（形狀 N×CHW），輸出為 O_{n,m}（N×M），運算即升級為**矩陣–矩陣乘**：

$$O_{n,m} = I_{n,chw} \times F_{m,chw}$$

投影片明確指出：「對 Einsum 而言，秩的順序無關緊要」——`O_{n,m} = I_{n,chw} × F_{m,chw}` 與實驗室常用記法 `Z_{m,n} = A_{m,k} × B_{k,n}` 描述的是同一個抽象運算，只是秩命名慣例不同。

### 卷積層（CONV）作為 Einsum

完整的 2-D CONV 層，有批次 N、M 個輸出通道、C 個輸入通道、輸出空間大小 P×Q、濾波器空間大小 R×S，其 Einsum 為：

$$O_{n,m,p,q} = B_m + I_{n,c,\,U \times p+r,\,U \times q+s} \times F_{m,c,r,s}$$

（U 為步幅（stride）；B 為偏置（bias），為簡化起見忽略。）關鍵的結構事實：輸入以*組合表達式* `U×p+r`（高度）和 `U×q+s`（寬度）為索引。這就是將 CONV 與 FC 區分開來的**托普利茲關係**——在 FC 中每個輸出都看到每個輸入；在 CONV 中每個輸出只看到由濾波器位置決定的輸入局部 patch。

![卷積 Einsum——完整 CONV 層的代數記法與 Einsum 記法](../../assets/L04/L04-p43-convolution-einsum.png)

### 將 CONV 轉換為矩陣乘法（im2col / 托普利茲）

從 CONV 到矩陣乘的轉換分三步進行：

1. **提取托普利茲結構：** 定義托普利茲張量 `T_{c,p,q,r,s} = I_{c,p+r,q+s}`，預先填充輸入的每個 patch。
2. **扁平化秩：** 將 `(c,r,s)` 壓縮為 `crs`，將 `(p,q)` 壓縮為 `pq`。
3. **得到的 Einsum 為：**

$$O_{m,pq} = T_{pq,crs} \times F_{m,crs}$$

這是一個矩陣乘法：濾波器（M×CRS）× 托普利茲輸入（PQ×CRS）= 輸出（M×PQ）。

![CONV 層化歸矩陣乘——O_{m,pq} = T_{pq,crs} × F_{m,crs}，P=H-R+1，Q=W-S+1](../../assets/L04/L04-p167-conv-to-matmul.png)

加入批次 N：`O_{m,pqn} = T_{pqn,crs} × F_{m,crs}`——仍然是矩陣乘法。

### 小結洞察

「Summary」投影片（PDF 頁 172 / 邏輯投影片 153）以簡潔的文字陳述要點：

- 我們已展示 FC 和 CONV 如何轉換為矩陣乘法。
- 有**許多方式**計算這個矩陣乘法（MACs 可以用任意順序執行並得到相同輸出）。
- 然而，**處理順序影響能耗和延遲等硬體量測指標**——具體而言，處理順序決定了必須搬移多少資料。

這就是從 Einsum（算什麼）到映射問題（怎麼算）的橋梁，而映射問題主導了 L05–L06。

> **為什麼重要：** 任何能執行通用矩陣乘法的硬體都能執行 FC 和 CONV。但它「如何」執行那個矩陣乘——最內層迴圈跑哪個維度、哪些資料被快取在何處——決定了能耗是接近暫存器檔（RF）的下限，還是趨近 DRAM 的天花板。

## 第四章 — 從卷積到注意力機制

> *投影片：L04-174 … L04-178*

在推導 Transformer Einsum 之前，本講先提供動機：為什麼某些任務更偏好注意力而非卷積？

### 卷積 vs. 注意力機制

| | 卷積（Convolution） | 注意力（Attention） |
|---|---|---|
| 依賴關係範圍 | 僅局部鄰域（R×S 濾波器） | 全域——任意 token 對任意 token |
| 感興趣區域 | 固定（由濾波器位置決定） | 動態（由查詢 query 決定） |
| 代價 | 輸入大小的線性函數 | 序列長度的二次函數 |

引自 Vaswani 等人（NeurIPS 2017）的表述：**「允許對全域依賴關係進行建模，而不受其距離影響。」** 注意力把計算資源集中在輸入中最具資訊量的部分，而非固定的空間視窗。

### Transformer——注意力的棲身之所

Transformer 架構（Vaswani，NeurIPS 2017）由注意力模組構建而成。它已被應用於：

- 自然語言處理（GPT-3，Brown 等人，NeurIPS 2020）
- 音訊（AST，Gong 等人，Interspeech 2021）
- 視覺（ViT，Dosovitskiy 等人，ICLR 2021）

輸入被分割成**token**（文字對應詞、視覺對應圖像 patch、音訊對應頻譜 patch）。注意力機制讓每個 token 能動態為所有其他 token 加權。

![卷積 vs. 注意力——局部固定感受野 vs. 全域動態依賴](../../assets/L04/L04-p175-conv-vs-attention.png)

## 第五章 — 以 Einsum 表達自注意力

> *投影片：L04-179 … L04-198*

本章推導自注意力的完整 Einsum 鏈。每個步驟是一個 Einsum；合在一起構成完整的計算。

### 圖示慣例

投影片使用一致的視覺語言：
- **圓角框** → 張量（如 A、Q、K、V、Z）
- **六邊形框** → 運算（×、softmax）
- **垂直線紋框** → 投影（乘以權重矩陣 W）

### 步驟 0 — 輸入嵌入（Input Embedding）

將 one-hot 的原始 token IR（形狀 M×C）轉換為稠密嵌入向量 I（形狀 M×D）：

$$I_{m,d} = IR_{m,c} \times WI_{c,d}$$

（只適用於第一層；後續層直接接收上一層的輸出。）

### 步驟 1 — 投影到查詢（Q）與鍵（K）

將嵌入從 D 空間投影到 E 空間，分別生成 Q 和 K：

$$Q_{m,e} = I_{m,d} \times WQ_{d,e}$$
$$K_{m,e} = I_{m,d} \times WK_{d,e}$$

兩者均為形狀 (M×D) × (D×E) → (M×E) 的矩陣乘法。WQ 和 WK 是**靜態張量（static tensors）**（在推論時對所有輸入共享）。

### 步驟 2 — 計算 QK（softmax 前的注意力分數）

$$QK_{m,p} = Q_{p,e} \times K_{m,e}$$

注意：p 是 m 的別名（兩者均索引序列長度 M）。這計算了每對查詢向量與鍵向量之間的點積——一個 (M×E) × (E×M) → (M×M) 的矩陣乘法。代價為 O(M² E)。

### 步驟 3 — Softmax，形成注意力權重 A

$$SN_{m,p} = \exp(QK_{m,p})$$
$$SD_p = \sum_m SN_{m,p}$$
$$A_{m,p} = SN_{m,p} / SD_p$$

Softmax 將 QK 的每一行正規化為對 M 個鍵的概率分佈。A 是**注意力張量（attention tensor）**（形狀 M×M）。

### 步驟 4 — 投影到值（V）

$$V_{m,f} = I_{m,d} \times WV_{d,f}$$

將嵌入從 D 空間投影到 F 空間（形狀 M×F）。WV 也是靜態張量。

### 步驟 5 — 值的加權求和（AV）

$$AV_{p,f} = A_{m,p} \times V_{m,f}$$

用注意力權重對值向量做加權乘積。形狀：(M×M) × (M×F) → (M×F)。這正是「注意力」所在——它為每個輸出位置取所有值向量的加權組合。

### 步驟 6 — 輸出投影

$$Z_{p,g} = AV_{p,f} \times WZ_{f,g}$$

從 F 空間投影到 G 空間（輸出嵌入維度，通常等於 D）。形狀：(M×F) × (F×G) → (M×G)。WZ 是靜態張量。

![基本自注意力編碼器計算——所有張量與運算的完整資料流圖](../../assets/L04/L04-p182-basic-self-attention.png)

### 完整串聯（Full Cascade）

將全部六個步驟合在一起：

![完整注意力串聯——一張投影片呈現所有 Einsum](../../assets/L04/L04-p189-full-cascade.png)

五個權重矩陣 WI、WQ、WK、WV、WZ 是**靜態**的（不隨輸入改變）。所有其他張量（I、Q、K、V、QK、SN、SD、A、AV、Z）都是**動態**的（對每個輸入序列重新計算）。

### 計算屬性

- **矩陣乘法總數：** 5 個（WI、WQ、WK 用於輸入投影；QK 點積；AV 加權求和）+ 1 個輸出投影（WZ）= 共 6 個。
- **可並行性：** Q、K、V 的投影彼此獨立，可並行執行。
- **順序依賴：** QK 必須先於 AV（AV 依賴由 QK 導出的 A）；投影必須先於核心注意力計算。
- **二次方縮放：** QK 矩陣乘法為 M×E × E×M = O(M²E)；AV 乘法為 M×M × M×F = O(M²F)。兩者均以序列長度 M 的**二次方縮放**——這是長上下文場景中的主導成本。

### 批次注意力（Batched Attention）

加入批次秩 B 很直接：每個張量都增加一個前置的 B 維度，所有 Einsum 都增加下標 `b`。權重張量 WQ、WK、WV、WZ 保持二維（在批次元素間共享）。

### 多頭注意力（Multi-Head Attention, MHA）

不同的注意力頭（head）可以學習關注不同面向（例如局部 vs. 全域依賴），方法是每個頭使用獨立的 Q/K/V 投影：

$$K_{b,h,m,e} = I_{b,m,d} \times WK_{d,h,e}$$
$$Q_{b,h,m,e} = I_{b,m,d} \times WQ_{d,h,e}$$
$$V_{b,h,m,f} = I_{b,m,d} \times WV_{d,h,f}$$

秩 H（頭數）被加入所有張量。計算各頭的注意力輸出後，沿 H 和 F 維度**串接（concatenate）**，再進行最終投影：

$$C_{b,p,h \times F + f} = AV_{b,h,p,f}$$
$$Z_{b,p,d} = C_{b,p,f} \times WZ_{g,d}$$

![多頭注意力——所有各頭 Einsum、最終串接與輸出投影](../../assets/L04/L04-p194-multi-head-attention.png)

### 秩名詞彙表（投影片 195–198）

| 秩 | 含義 |
|---|---|
| **M** | 序列長度（自注意力中的查詢/鍵/值） |
| **P** | M 的別名（QK 的輸出序列位置） |
| **C** | 詞典大小（詞彙量） |
| **D** | 全域嵌入（d_model） |
| **E** | Q/K 的局部嵌入（d_k） |
| **F** | V 的局部嵌入（d_v） |
| **G** | 輸出嵌入 |
| **B** | 批次大小 |
| **H** | 多頭注意力的頭數 |

> **為什麼重要：** 以 Einsum 表達注意力，使其結構與矩陣乘法一樣透明。針對通用矩陣乘法（GEMM）可用的所有最佳化——分塊、管線化、稀疏性利用——在此同樣潛在適用。這正是 L01 案例研究中 FuseMax 等注意力加速器的設計基礎。

## 獨立學習指南（Standalone Study Guide）

### 進入下一講前必須掌握

- 將一般迴圈巢轉成 Einsum，並能由 Einsum 還原迴圈巢。
- 把切分（partitioning）與展平（flattening）理解為秩轉換，而不是改變數學張量本身。
- 推導 FC 與 CONV 如何表示成 matrix-vector 或 matrix-matrix operation。
- 說明 attention 具有 token 間二次互動，而 convolution 具有固定局部 receptive field。

### 自我檢核問題

1. Einsum 右側重複出現的索引代表什麼？
2. 卷積轉成矩陣乘法時，Toeplitz-like 結構從哪裡來？
3. Softmax 為什麼是 attention pipeline 中主要的非線性操作？它位在 Einsum 串接的哪個位置？

### 練習

1. 為 `Z[m,n] = A[m,k] * B[k,n]` 寫出迴圈巢，並標出自由秩與 reduction rank。
2. 將 2-D 座標 `(r,s)` 展平成單一秩，並展示如何還原 `r` 與 `s`。
3. 畫出單一 attention head 的資料相依鏈，包含 Q、K、V、attention weights 與 output projection。

### 常見誤區

- 以為 flattening 在硬體上免費。它會改變位址產生方式，也可能改變 locality。
- 在 FC、CONV、attention 範例間切換時混淆 rank names。
- 把 attention 當成「只是 matmul」；動態 score matrix 與 softmax 會改變資料流限制。

## 關鍵詞彙（Key Terms）

| 詞彙 | 說明 |
|---|---|
| **Einsum** | 一種緊湊的代數記法，用於 DNN 運算：指定運算元、秩與算子，但不規定求值順序。 |
| **ODE（Einsum 操作定義）** | 定義：遍歷迭代空間中的所有點；在每個點計算右側並賦值/歸約到左側。 |
| **秩變數（rank variable）** | 張量某一維度的索引（如 m、n、k）。 |
| **秩名（rank name）** | 維度的語義名稱（如「M」、「K」）；與秩變數字母無關。 |
| **秩形狀（rank shape）** | 維度的大小（範圍）。 |
| **收縮秩（contracted rank）** | 只出現在右側的秩；其值被加總（歸約）。 |
| **未收縮秩（uncontracted rank）** | 同時出現在兩側的秩；每個值產生一個獨立的輸出元素。 |
| **分割（Partitioning）** | 將一個秩拆分為兩個（如 i → i1, i0，其中 i = i1×I0 + i0）。總是新增一個秩。 |
| **扁平化（Flattening）** | 將兩個或多個秩合併為一個複合索引。總是移除一個秩。 |
| **FC 層（全連接層）** | Einsum 為 O_m = I_{c,h,w} × F_{m,c,h,w}；扁平化後等價於矩陣–向量乘。 |
| **CONV 層（卷積層）** | Einsum 為 O_{n,m,p,q} = I_{n,c,Up+r,Uq+s} × F_{m,c,r,s}；托普利茲轉換後等價於矩陣乘法。 |
| **托普利茲張量 / im2col** | 預先填充的 patch 張量 T_{pq,crs} = I_{c,p+r,q+s}，將 CONV 轉換為矩陣乘法。 |
| **注意力（Attention）** | 一種動態選取感興趣區域的機制；「允許對全域依賴關係進行建模」（Vaswani 2017）。 |
| **Token** | Transformer 的基本輸入單元（詞、圖像 patch、頻譜 patch）。 |
| **Q、K、V** | 查詢（Query）、鍵（Key）、值（Value）——自注意力中輸入的三個線性投影。 |
| **注意力張量 A** | 所有查詢與鍵之間點積分數的 M×M softmax 正規化矩陣。 |
| **自注意力（Self-Attention）** | Q、K、V 均來自同一輸入序列的注意力。 |
| **多頭注意力（MHA）** | 有 H 個獨立 Q/K/V 頭的注意力，使模型能對輸入形成多個並行「視角」。 |
| **二次方縮放（Quadratic scaling）** | QK 與 AV 矩陣乘法均以序列長度 M 的二次方增長。 |
| **靜態張量（Static tensors）** | 不隨每次輸入改變的權重矩陣（WQ、WK、WV、WZ）。 |
| **動態張量（Dynamic tensors）** | 對每個輸入重新計算的所有中間激活（Q、K、V、QK、A、AV、Z）。 |

## 重點回顧（Takeaways）

- **Einsum** 有精確的操作語義（ODE）：遍歷迭代空間、計算、賦值/歸約。它完全**與執行順序無關**。
- 最重要的結構概念：只出現在右側的秩是**收縮秩**（被加總）；同時出現在兩側的秩是**未收縮秩**（產生獨立的輸出元素）。
- **分割**與**扁平化**互為逆運算：它們改變 Einsum 的代數形式，而不改變計算結果，是硬體分塊的代數基礎。
- **FC**（扁平化後）和 **CONV**（經由托普利茲/im2col 轉換後）都化歸為**矩陣乘法**——DNN 加速器設計的通用基本操作。
- CONV 化歸矩陣乘的過程揭示了「處理順序影響硬體量測指標」的原因：相同的矩陣乘結果可以透過截然不同的資料搬移模式得到，取決於哪個維度在最內層迴圈。
- Transformer 中的**自注意力**是六個 Einsum 步驟的序列；其中五個是矩陣乘法，外加一個 softmax。共 6 個矩陣乘法（含輸出投影），且對序列長度具有二次方依賴。
- **多頭注意力**為所有各頭張量新增秩 H，並在最終投影前串接各頭輸出。
- 以 Einsum 表達注意力，使其與 CONV 和 FC 站在同等的地位：相同的映射、分塊與稀疏性分析工具均適用。

## 與後續講次的連結（Connections）

- **映射與資料流（L05–L06）：** Einsum 是 TeAAL 金字塔映射層（Mapping layer）的「Compute」輸入。L05–L06 列舉 Einsum 各秩上所有合法的迴圈排序，並衡量其資料搬移影響。
- **稀疏性（L07–L10）：** 注意力權重矩陣（A）在 softmax 閾值化後常常稀疏；CONV/FC 中的激活也有結構化稀疏性。Einsum 的 Format 層（稀疏表示）直接影響收縮秩的遍歷方式。
- **進階技術／精度（L11–L12）：** 降低精度的算術運算按各 Einsum 的運算元分別施加；Einsum 的結構決定了哪些近似是安全的。
- **FuseMax（L01 案例研究）：** FuseMax 利用注意力的 Einsum 結構，跨越 softmax 邊界融合運算，消除中間 DRAM 流量。注意力的 Einsum——特別是 `AV_{p,f} = A_{m,p} × V_{m,f}`——正是該最佳化的直接目標。
